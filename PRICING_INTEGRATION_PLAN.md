# 2. ОПЛАТА

## Рекомендованный стек

### Основной: Stripe Checkout
- Карты (Visa, Mastercard, AmEx)
- Apple Pay / Google Pay
- SEPA, iDEAL (Европа)
- Автоматические рекарринги
- Годовые/месячные подписки нативно
- Портал самообслуживания (смена карты, история)

### Дополнительно: TON (Telegram)
- Крипто-аудитория
- Без банковских ограничений
- Для Enterprise — по запросу

### НЕ рекомендую: Telegram Payments
- Ограниченные провайдеры
- Нет нормальных рекаррингов
- Сложная настройка

## Флоу оплаты через Stripe

```
Пользователь нажимает [Оплатить $299/мес]
        ↓
Бот отправляет ссылку на Stripe Checkout
        ↓
stripe.com/pay/... → пользователь вводит карту
        ↓
Webhook → stripe/webhook → активация в БД
        ↓
Бот: "✅ Оплата прошла! Разворачиваю агента..."
        ↓
Деплой агента → отправка данных @YourBot
```

## Месяц/Год toggle при оплате

Реализация: два разных Price ID в Stripe для каждого тарифа.

```js
const prices = {
  solo: {
    monthly: 'price_solo_monthly_299',
    yearly: 'price_solo_yearly_2509'
  },
  team: {
    monthly: 'price_team_monthly_599', 
    yearly: 'price_team_yearly_5029'
  },
  // ...
}
```

В боте — кнопки переключения прямо в сообщении с планом.

## Апгрейд/даунгрейд

**Апгрейд (Solo → Team):**
- Stripe proration — автоматический пересчёт
- Пользователь платит разницу за оставшиеся дни
- Новые агенты появляются сразу

**Даунгрейд (Team → Solo):**
- Остаётся на текущем плане до конца периода
- С нового периода — Solo
- Лишние агенты архивируются (не удаляются)

## Промокоды
```js
// Stripe Coupon API
const coupon = await stripe.coupons.create({
  percent_off: 20,
  duration: 'once', // или 'repeating', 'forever'
  name: 'LAUNCH20'
});
```

Форма ввода промокода — на Stripe Checkout (нативно).

## Политика возвратов
- Starter ($79/7 дней): no refund после активации агента
- Solo/Team: возврат в течение 24 часов если агент не задеплоился
- Business/Enterprise: обсуждается индивидуально
- При техническом сбое с нашей стороны — full refund

# 4. МЕТРИКИ И АНАЛИТИКА

## Ключевые события для логирования

```js
// При каждом событии отправлять в БД + опционально в аналитику

events = [
  'trial_started',         // user_id, date
  'trial_day3_active',     // использовал агента в первые 3 дня
  'trial_converted',       // trial → платный план
  'trial_expired',         // не конвертировался
  'plan_purchased',        // plan, billing_period (monthly/yearly), amount
  'plan_upgraded',         // from_plan, to_plan
  'plan_downgraded',       // from_plan, to_plan
  'plan_cancelled',        // plan, reason (optional)
  'payment_failed',        // plan, retry_count
  'limit_hit',             // тип лимита (tokens/agents/features)
  'upgrade_prompt_shown',  // какой триггер, какой план предложили
  'upgrade_prompt_clicked',// конверсия промпта
  'feature_unlocked',      // какая фича разблокирована при апгрейде
  'agent_deployed',        // plan, time_to_deploy
  'support_contacted',     // план пользователя, тип запроса
]
```
